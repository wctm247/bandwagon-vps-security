## 第三层安全：账号会不会被封，IP 会不会被墙

这是问得最多的一类，也是最容易说清楚的。

**账号封禁的原因**几乎都是违规操作，不是商家随机抽奖封人。搬瓦工的服务条款里明确列了禁止的行为，大意包括：

- 发垃圾邮件（大批量 SMTP 连接）
- 对外发起 DDoS 或类似攻击
- 传播恶意软件
- 分发没有授权的版权内容
- BitTorrent 相关活动

正规使用——比如搭建个人网站、跑开发环境、学 Linux、部署自己的服务——完全不会触碰这些红线。

**IP 被封**这件事，主要和你所在地区的网络环境有关，跟搬瓦工平台本身关系不大。搬瓦工提供的 CN2 GIA、CN2 GT、CMIN2 等线路都是正经的商业线路，稳定性比普通 BGP 线路好得多。

说说我自己的情况。洛杉矶 DC6 的 IP 用了差不多一年半，没有出现过被封的问题。断线大多是因为路由波动或者本地网络，基本上等几分钟或者换个连接方式就恢复了。

## 第四层安全：VPS 系统本身的安全该怎么配

这块是搬瓦工安全话题里讲得最少、但实际上最重要的一块——VPS 本身是一台暴露在公网上的服务器，默认配置不做任何安全加固，是真的容易被扫到漏洞然后被人利用。

搬瓦工提供的 KiwiVM 面板里有一些基础安全工具，比如：

1. 登录 KiwiVM 后台后，SSH 端口默认已经是随机的高位端口（不是 22），这个**不要改回 22**，改了反而是在给自己挖坑
2. Root 密码初始是随机生成的，用完后建议改成长随机密码，记在密码管理器里，**不要用有规律的密码**
3. 建议配置 SSH 密钥登录，把密码登录禁掉——这是减少暴力破解的核心操作
4. 快照功能：KiwiVM 里能创建全盘快照，在折腾系统配置之前先存档，哪怕搞坏了也能一键回滚
5. 搬瓦工会自动定期备份整个 VPS 状态，但自动备份间隔不短，关键数据建议额外做手动备份

另外，每隔一段时间留意一下 KiwiVM 上的流量监控。如果某天流量异常飙升但自己并没有大量传输数据，大概率是被人入侵拿去当跳板了，这时候及时重装系统是最干净的处理方式。

## 常见问题 Q&A

**Q：搬瓦工VPS安全吗，数据会不会丢失？**  
A：商家层面的安全没有问题，IT7 运营超过二十年，服务连续性有保障。数据丢失的风险更多来自自身操作失误，建议定期用 KiwiVM 创建快照备份，重要数据另存一份到本地。

**Q：搬瓦工IP容易被封吗？**  
A：正规用途下基本不会。CN2 GIA 和 CMIN2 线路的 IP 稳定性很好，遇到的问题更多是短暂路由波动，不是真正的封禁。触碰服务条款（比如发垃圾邮件）才会导致账号被封。

**Q：搬瓦工的 KiwiVM 面板安全吗？**  
A：KiwiVM 是搬瓦工自研的管理面板，登录有独立密码，和 VPS 的 root 密码是分开的。建议设置独立强密码，不要和 VPS 系统密码一样。

**Q：搬瓦工VPS被黑了怎么办？**  
A：最干净的处理方式是直接在 KiwiVM 里重装系统，几分钟搞定。如果有之前的快照，可以直接回滚到被黑之前的状态。事后建议重新配置 SSH 密钥登录，关掉密码认证。

**Q：搬瓦工支持支付宝吗？**  
A：支持。国内用户直接扫码付款，没有外币汇率问题，到账后马上开通服务。
